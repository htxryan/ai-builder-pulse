# site/

Static brochure site deployed at `pulse.ryanhenderson.dev`.

## Files

| File | Purpose |
|---|---|
| `index.html` | Landing page — hero, latest-issue preview, signup form |
| `styles.css` | Synthwave CSS design system — shared by index, archive pages, and per-issue pages |
| `app.js` | Compiled JS bundle for the brochure (latest-issue fetch, signup handler) |
| `app.d.ts` | TypeScript declarations for `app.js` |
| `favicon.svg` | Site favicon |
| `og-image.svg` | Open Graph / Twitter Card image |
| `CNAME` | Custom domain (`pulse.ryanhenderson.dev`) |
| `buttondown-theme.css` | Custom CSS for Buttondown's hosted pages — see below |

Archive pages (`/archive/`, per-issue) are generated by `scripts/lib/archivePage.ts` and `scripts/lib/issuePage.ts` and share `styles.css`.

## Applying the Buttondown theme

`buttondown-theme.css` overrides Buttondown's default light theme with the synthwave palette to match this site. It targets the hosted archive index and individual email web-view pages at `buttondown.com/ai-builder-pulse`.

**How to apply / update:**

1. Copy the full contents of `site/buttondown-theme.css`.
2. Go to [buttondown.com/settings/design](https://buttondown.com/settings/design).
3. Paste into the **Custom CSS** textarea and save.

`buttondown-theme.css` is the source of truth. Every time it changes, repeat the paste-and-save step above — Buttondown has no git integration.

**What it covers:**

- Global dark background (`#07031a`) replacing Buttondown's default `#FAFAFA`.
- Header nav bar: magenta chrome-gradient wordmark, monospace type, cyan hover state.
- Archive index: row-based list matching the custom site's archive layout.
- Email web-view: cyan section headings, magenta item links, mono code blocks, blockquote styling.
- Subscribe form: glowing magenta border card, magenta gradient CTA button.
- CRT scanlines overlay.
- `prefers-reduced-motion` support.

**What it does NOT cover:**

- Email HTML body (what Gmail/Apple Mail render) — that's controlled by `src/renderer/renderer.ts`.
- The subscribe/confirm/unsubscribe transactional pages (Buttondown system pages, not customisable via CSS alone).
